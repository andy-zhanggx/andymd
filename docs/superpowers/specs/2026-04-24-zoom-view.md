# Spec: View → Zoom (Markdown-adapted)

**Date:** 2026-04-24
**Branch:** `feature/zoom-view` (new, off `feature/rename-andymd`)
**Status:** Draft — awaiting Andy's approval

## Motivation

Typora/Preview-style View menu needs zoom controls. The reference screenshot lists 11 PDF-era modes; most (Marquee zoom, Loupe, Reflow, etc.) don't apply to a flowing Markdown document. We implement only the modes that carry meaning for a long-form text editor, following Typora's own conventions.

## Scope — features to implement

| Menu item | Shortcut | Behavior |
|---|---|---|
| **Zoom In** | `⌘+` (and `⌘=`) | Increase `zoomLevel` by 10 (capped at 300) |
| **Zoom Out** | `⌘-` | Decrease `zoomLevel` by 10 (floored at 50) |
| **Actual Size** | `⌘0` | Reset `zoomLevel` to 100 |
| **Zoom to…** | `⌘Y` | Prompt for a percentage (50–300); apply |
| **Fit to Width** | `⌘2` | Toggle `fitToWidth` boolean. ON = drop `max-width: 740px`, full-window width with 64px side padding. OFF = reading width (default). Checkmark reflects state. |

Explicitly **not implementing** (no markdown analog): Fit height, Fit visible content, Zoom to page level, Marquee zoom, Dynamic zoom, Pan & zoom, Loupe zoom, Reflow.

## Design

### Rendering: CSS `zoom` property

Apply `zoom: <level>%` on `.editor-container`. `zoom` is supported in both WebKit and Blink (Tauri's targets), scales layout + click coordinates correctly, and avoids the reflow pain of `transform: scale()`. Only the editor content scales; sidebar, menu, and status bar stay at 100%.

### `fitToWidth` toggle

Add a `.editor-container.fit-width` modifier class:
- Default (`.editor-container`): `max-width: 740px; margin: 0 auto; padding: 32px 24px;` (current behavior)
- Fit-width (`.editor-container.fit-width`): `max-width: none; margin: 0; padding: 32px 64px;`

### Persistence

Extend `AppConfig` (`src/types.ts`) and `DEFAULT_CONFIG`:
```ts
zoomLevel: number;   // 50..300, default 100
fitToWidth: boolean; // default false
```
Both persist through the existing `configService.save()` path. No Rust schema change needed — config is a JSON blob on the Rust side.

### Menu wiring (`src-tauri/src/menu.rs`)

Add to the View submenu, after `toggle-sidebar`:
- Separator
- `zoom-in` — "Zoom In", accelerator `CmdOrCtrl++`
- `zoom-out` — "Zoom Out", accelerator `CmdOrCtrl+-`
- `zoom-reset` — "Actual Size", accelerator `CmdOrCtrl+0`
- `zoom-to` — "Zoom to…", accelerator `CmdOrCtrl+Y`
- Separator
- `fit-to-width` — "Fit to Width", accelerator `CmdOrCtrl+2` (CheckMenuItem, bound to `fitToWidth`)

All emit through the existing `"menu"` event channel.

### Frontend dispatch (`src/hooks/useShortcuts.ts`)

Add cases in `handleMenuAction()`:
- `zoom-in` / `zoom-out` / `zoom-reset` → update `configStore.zoomLevel`
- `zoom-to` → `window.prompt("Zoom level (50–300):", String(zoomLevel))`, validate, update
- `fit-to-width` → toggle `configStore.fitToWidth`

### Application point (`src/components/Editor/MarkdownEditor.tsx`)

Read `zoomLevel` and `fitToWidth` from the store. Apply:
```tsx
<div
  className={`editor-container${fitToWidth ? ' fit-width' : ''}`}
  style={{ zoom: `${zoomLevel}%` }}
  ref={ref}
/>
```
Remove hard-coded `maxWidth/margin/padding` inline styles — move them into `editor-styles.css` where `.fit-width` can override.

### CheckMenuItem sync for Fit to Width

On config load and after toggle, the Rust side has no reactive hook to the menu item. MVP: accept that the checkmark only reflects state on click (Tauri's CheckMenuItem updates on user interaction). If this is visibly wrong, revisit post-MVP.

- `src-tauri/src/menu.rs` — add 5 items + 2 separators

## Verification

1. `pnpm tauri dev` launches; no TS/Rust errors.
2. View menu shows the 5 new items with the listed accelerators.
3. `⌘+` three times → content visibly larger; `⌘-` three times → back to 100%; `⌘0` resets from any state.
4. `⌘Y` prompts; entering `150` applies; entering `9999` or `abc` is rejected (no crash, no change).
5. `⌘2` toggles reading-width ↔ full-window width; sidebar/menu/statusbar do not resize.
6. Restart app: `zoomLevel` and `fitToWidth` persist.
7. Edge: zoom is clamped to [50, 300] — `⌘-` at 50 is a no-op; `⌘+` at 300 is a no-op.

## Out of scope

- Persisting per-document zoom (global only).
- Zoom the entire window chrome (only editor content zooms).
- Animated transitions.
- Touchpad pinch-zoom gesture.
